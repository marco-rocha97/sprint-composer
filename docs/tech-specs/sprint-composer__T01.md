## Contracts

### Transcript file format (`fixtures/transcript.txt`)

Plain text. The file opens with a header block, then a separator line, then the body. Each segment in the body is a paragraph of consecutive non-blank lines. Blank lines separate segments.

```
day: <integer 1ŌĆō15>
phase: <one of: Discovery | Setup | Simulation | Go-live>
participants: [<Name>, <Name>, ...]
---
<body: free-text dialogue>
```

**Header parsing rules** (enforced by T05 ŌĆö defined here so all tasks use the same contract):

| Field | Type | Valid values |
|---|---|---|
| `day` | integer | 1ŌĆō15 inclusive |
| `phase` | string | exactly `Discovery`, `Setup`, `Simulation`, or `Go-live` (case-sensitive) |
| `participants` | list | non-empty; bracket syntax `[A, B, C]` |

**Demo transcript scenario:** patient portal integration meeting for a healthcare provider, day 10, Simulation phase.

**Required segment inventory** (minimum ŌĆö additional segments are allowed):

| Segment ID | Expected Layer-1 type | Topic / nature |
|---|---|---|
| S01 | `firm_request` | SSO / Active Directory integration |
| S02 | `firm_request` | Push notification for appointment reminders |
| S03 | `firm_request` | New reporting dashboard *(out-of-phase ŌĆö new scope introduced on day 10 Simulation; satisfies demo criterion 3)* |
| S04 | `firm_request` | IoT glucose monitor proprietary API integration *(no reference bank match ŌåÆ LOW confidence / "estimate not available"; satisfies demo criterion 2)* |
| S05 | `latent_request` | Nursing staff pain: switching between three separate systems daily (unresolved pain, not a stated request) |
| S06 | `decision` | Agreement reached: staging environment to be used for UAT |
| S07 | `open_question` | Who is responsible for HIPAA compliance sign-off on the integration layer? |
| S08 | `noise` | Off-topic social remark (does not pertain to the project) |

### Taxonomy template (`fixtures/taxonomy_template.json`)

```json
{
  "transcript": "fixtures/transcript.txt",
  "segments": [
    {
      "segment_id": "S01",
      "excerpt": "<verbatim opening phrase of the segment, Ōēź20 chars>",
      "expected_type": "<firm_request | latent_request | decision | open_question | noise>",
      "notes": "<optional human-readable explanation of why this type>"
    }
  ]
}
```

**Requirements:**
- One entry per body segment in `transcript.txt` ŌĆö no segment may be missing.
- At least one entry for each of the five Layer-1 types.
- `excerpt` must be a verbatim substring of the corresponding transcript segment (copy-paste, no paraphrase).
- `segment_id` format: `S` followed by a zero-padded two-digit integer (`S01`ŌĆ”`S08`).

## Trade-offs and Rejected Alternatives

**Decision: JSON for structured fixtures (not YAML)**
- **Rejected alternative:** YAML (`.yaml` files, `yaml.safe_load`)
- **Reason:** YAML requires PyYAML ŌĆö a new runtime dependency with no other role in the repo. Python's stdlib `json` handles this without installation. JSON is also consistent with the JSON sibling output specified for T05.
- **Source:** `rules/dependencies.md` ("no new library without discussion"); `rules/anti-patterns.md` ("reuse over install").

**Decision: `fixtures/` at project root (not `tests/fixtures/`)**
- **Rejected alternative:** `tests/fixtures/`
- **Reason:** The transcript is a first-class demo input ŌĆö `sprint-composer run fixtures/transcript.txt` appears in the demo script and README. Placing it under `tests/` implies it is test-only and makes the demo command less legible.
- **Source:** SPEC Experience Design ("FDE runs the CLI: `sprint-composer run <transcript.txt>`"); user decision.

**Decision: T01 bundles project scaffolding**
- **Rejected alternative:** Scaffolding as an implicit prereq (done manually before starting T01)
- **Reason:** No test can run without `pyproject.toml` + `uv.lock`. Delivering fixture files without a working test runner leaves T01 unverifiable. Bundling produces one coherent, shippable commit.
- **Source:** User decision; T01 has no predecessor task.

**Decision: CLI entry point declared in T01's `pyproject.toml` as a placeholder**
- **Rejected alternative:** Add the entry point only in T05
- **Reason:** `uv sync` installs the package in editable mode; the `[project.scripts]` binding must exist from day one so the `sprint-composer` command resolves in all tasks. T05 creates the implementation; T01 creates the binding.
- **Source:** Python packaging convention; `uv` editable install behavior.

## Risks and Mitigations

| Risk | Impact | Mitigation |
|---|---|---|
| Taxonomy template falls out of sync with transcript | Layer 1 graded against wrong expected types | Test asserts every body segment has a matching `segment_id` in the template (bidirectional) |
| Reference bank has no gap (all requests match) | Demo criterion 2 unachievable downstream | Test asserts no task keyword-overlaps with the IoT glucose monitor topic |
| Header format ambiguity causes T05 parser inconsistency | Parser breaks on the fixture file | Header format is fully specified in this document; T05 references this spec |
| `uv.lock` not committed | Reproducibility broken across machines | `.gitignore` excludes `.venv/` only; `uv.lock` is committed per `rules/dependencies.md` |

## Testing Plan

**File:** `tests/test_fixtures.py`

Uses stdlib only (`json`, `pathlib`, `re`). No pytest plugins. Every test is a structural assertion on the static fixture files.

**Transcript ŌĆö header (4 tests):**
- `test_header_has_required_fields` ŌĆö parsed header contains `day`, `phase`, `participants`
- `test_header_day_is_integer_in_range` ŌĆö `day` is an integer in [1, 15]
- `test_header_phase_is_valid` ŌĆö `phase` is exactly one of `Discovery`, `Setup`, `Simulation`, `Go-live`
- `test_header_participants_non_empty` ŌĆö `participants` is a non-empty list

**Transcript ŌĆö body (3 tests):**
- `test_at_least_one_segment_per_type` ŌĆö taxonomy template contains at least one entry for each of the five Layer-1 types
- `test_all_taxonomy_excerpts_in_transcript` ŌĆö every `excerpt` in the taxonomy is a verbatim substring of the transcript text
- `test_segment_count_matches_taxonomy` ŌĆö count of non-empty body segments equals count of taxonomy entries (nothing uncatalogued)

**Reference bank (4 tests):**
- `test_reference_bank_has_min_projects` ŌĆö at least 4 project entries
- `test_each_project_has_at_least_one_task` ŌĆö no project has an empty `tasks` list
- `test_required_task_fields_present` ŌĆö each task has `id`, `name`, `keywords`, `effort_days`, `effort_confidence`, `blockers`
- `test_effort_confidence_is_valid` ŌĆö `effort_confidence` in `{"HIGH", "MEDIUM", "LOW"}`

**Gap assertion (1 test):**
- `test_no_reference_match_for_glucose_api` ŌĆö no task across any project has keyword overlap (case-insensitive) with the terms `glucose`, `iot`, `glucometer`, or `device api`; this asserts the LOW-confidence path is exercisable

**Taxonomy template (2 tests):**
- `test_taxonomy_all_segment_ids_unique` ŌĆö no duplicate `segment_id`
- `test_taxonomy_types_valid` ŌĆö each `expected_type` is one of the five Layer-1 type strings

**Total: 14 tests.** No error-case tests ŌĆö these are static fixture files; structural correctness is the entire error surface.

> **Framework:** pytest (no plugins). Source: CLAUDE.md.

## Implementation Sequence

Each step is one cohesive commit:

1. **Project skeleton** ŌĆö `pyproject.toml` (dev deps + entry point placeholder), `.gitignore` (Python standard + `.env`, `.venv/`), `.env.example` (empty template), `src/sprint_composer/__init__.py` (empty), `tests/__init__.py` (empty)

3. **Pre-commit hooks** ŌĆö `.pre-commit-config.yaml` with `pre-commit-hooks` v5.0.0 (trailing-whitespace, end-of-file-fixer, check-merge-conflict, check-added-large-files) and `gitleaks` v8.18.4; run `pre-commit install`
4. **Author `fixtures/transcript.txt`** ŌĆö healthcare patient-portal meeting, day 10 Simulation phase, 8+ segments covering all five Layer-1 types; S03 is the out-of-phase request; S04 is the no-reference request
5. **Author `fixtures/reference_bank.json`** ŌĆö 4 projects with tasks covering SSO, push notifications, workflow consolidation, and reporting; deliberately no task matching the IoT glucose API
6. **Author `fixtures/taxonomy_template.json`** ŌĆö one entry per segment, verbatim excerpts copied from the transcript, one entry per Layer-1 type minimum
7. **Write `tests/test_fixtures.py`** ŌĆö all 14 tests
8. **Verify** ŌĆö `uv run pytest tests/test_fixtures.py -v` all green ┬Ę `uv run ruff check src/ tests/` clean ┬Ę `uv run mypy src/` clean
